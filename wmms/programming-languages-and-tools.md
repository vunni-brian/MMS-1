# 3.7 Design Techniques and Tools

The system design was prepared using tools that supported interface planning, system modelling, and implementation review. Draw.io was used to prepare system diagrams such as data flow diagrams and architecture diagrams. Figma was used for user interface planning and screen layout design. Visual Studio Code supported design validation during implementation by allowing the structure of frontend components, backend modules, and configuration files to be reviewed against the proposed system design.

The main design tools used were:

- Draw.io
- Figma
- Visual Studio Code

# 3.8 Programming Languages and Development Tools

The Market Stall Manager system was developed as a web-based application with a React and TypeScript frontend, a TypeScript-based Node.js backend, and a PostgreSQL database layer. Styling was implemented using CSS and Tailwind CSS, while SQL was used for database schema definition, migration, and data operations.

## Programming Languages and Technologies

- React
- TypeScript
- JavaScript
- CSS
- SQL
- PostgreSQL

## Development Tools and Frameworks

- Visual Studio Code
- Node.js
- npm
- Vite
- PostgreSQL
- Supabase
- Git
- Draw.io
- Figma

# 3.9 Testing Techniques and Methods

The system was tested using techniques that checked individual modules, interactions between frontend and backend services, complete user workflows, and acceptance against user requirements. Unit testing was used to verify small functions and isolated logic. Integration testing was used to confirm that modules such as authentication, vendor management, stall booking, payments, notifications, and reporting worked correctly together. System testing was used to validate complete end-to-end workflows across the application. User acceptance testing was used to confirm that the system met the operational needs of vendors, market managers, and officials.

The main testing techniques used were:

- Unit Testing
- Integration Testing
- System Testing
- User Acceptance Testing
